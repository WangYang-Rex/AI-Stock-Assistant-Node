## 一、为什么要用「多周期融合」而不是只看 K 线？

一句话总结：

> **趋势靠日线定方向，节奏靠短周期定时点**

单一 K 线的问题是：

* ✅ **日线 / 周线**

  * 优点：稳定、抗噪声、适合趋势
  * 缺点：**买卖点滞后**
* ❌ **分时 / 5 分钟**

  * 优点：灵敏
  * 缺点：**噪声极大，容易被假突破骗**

👉 多周期融合的核心思想是：

> **大周期 = 决定“能不能做”
> 小周期 = 决定“什么时候做”**

这在专业交易里叫：

* **Top-down Analysis**
* 或 **MTF（Multi-Timeframe）模型**

## 二、一个标准的「3 层多周期趋势模型」

我给你一个**非常实战、适合 A 股 ETF 的结构**：

### ① 大周期（趋势过滤层）——「值不值得参与」

**周期**：日线 / 周线
**作用**：只判断方向，不给买卖点

常用指标：

* 均线方向（20 / 60 / 120）
* ADX（是否有趋势）
* Donchian 通道（是否突破）

示例逻辑：

```text
只在以下条件成立时，才允许交易：
- 20 日 > 60 日 > 120 日
- 或 收盘价 > 60 日均线
```

👉 **这一步直接过滤掉 60%~70% 的震荡期**

---

### ② 中周期（结构确认层）——「趋势是否健康」

**周期**：30min / 60min / 日内 5 日线
**作用**：确认当前趋势有没有被破坏

常用方法：

* 中周期 EMA 结构
* MACD 是否死叉
* 回调是否“缩量不破位”

示例：

```text
- 价格回踩 20EMA 但未跌破
- MACD 未出现大级别背离
```

👉 这一层防止你：

* 追在趋势末端
* 买在“假强势”

---

### ③ 小周期（执行层）——「精确买卖点」

**周期**：5min / 15min / 分时
**作用**：只负责“下单时机”

常用信号：

* 放量突破
* 分时 VWAP 上穿
* 5min 均线金叉

示例：

```text
大周期看多 + 中周期回调完成
→ 小周期放量突破前高 → 执行买入
```

👉 **买卖点的“好看”，几乎全靠这一层**

## 三、为什么很多策略“故意不看分时”？

你前面那个策略只用 K 线，其实是**刻意的取舍**：

### 它隐含的假设是：

* 面向 **趋势跟随 / 波段**
* 追求 **稳健 + 可回测**
* 不追求最优买点，只追求**不犯大错**

分时线的问题在于：

* 📉 噪声太大
* 📉 容易过拟合
* 📉 回测和实盘一致性差

👉 **所以：**

* 新手策略 / 中长线策略 → 只用 K 线
* 成熟策略 / 资金效率优先 → 必须多周期

### 🎯 最终决策

```ts
if (trendAllowed && structureOK && entrySignal) {
  buy();
}
```

👉 这套结构：

* **不复杂**
* **可回测**
* **可视化非常好做（ECharts 多周期叠加）**
